# 📚 YOLO26-seg (Large) 파인튜닝 파이프라인 총정리 가이드

이 노트북은 새로운 데이터셋을 사용하여 YOLO26 인스턴스 세그멘테이션 모델을 학습(Fine-tuning)하고 배포 가능한 형태로 내보내는 전 과정을 자동화한 코드입니다. 초보자도 순서대로 실행만 하면 모델을 완성할 수 있도록 구성되어 있습니다.

### 📌 전체 진행 순서 및 각 단계별 역할

**1. 환경 설정 및 데이터셋 준비 (Step 1 ~ 5)**
- **환경 설정**: 모델 학습에 필요한 `ultralytics` 라이브러리와 필수 패키지들을 설치합니다.
- **데이터셋 다운로드**: 구글 공유 드라이브에 있는 압축된 데이터셋을 Colab 환경으로 가져와 압축을 풉니다.
- **데이터 설정 (YAML)**: 모델이 데이터를 인식할 수 있도록 클래스 정보와 경로가 담긴 `dataset.yaml` 파일을 자동 생성합니다.
- **라벨 검증 및 정제**: Segmentation 포맷에 맞지 않는 잘못된 라벨(Bounding Box 등)을 자동으로 필터링하여 학습 에러를 예방합니다.

**2. 모델 학습 (Step 6 ~ 7)**
이 노트북은 두 가지 학습 방식을 지원합니다:
- **Single-Stage (기본)**: 한 번에 전체 모델 가중치를 학습합니다. 빠르고 간편합니다.
- **2-Stage Fine-Tuning**: (성능 극대화 목적) 1단계에서 모델의 뼈대(Backbone)를 얼리고 머리 부분만 먼저 학습시킨 뒤, 2단계에서 전체를 아주 낮은 학습률로 미세 조정합니다. 데이터가 적을 때 특히 효과적입니다.

**3. 검증 및 테스트 (Step 8 ~ 10)**
- **Validation & Test**: 학습된 모델들(단일 학습, 2-Stage 학습 결과)을 검증/테스트 데이터셋으로 평가하여 정확도(mAP)를 수치로 비교합니다.
- **결과 시각화**: 실제 이미지에 모델이 예측한 마스크와 바운딩 박스를 덧씌워 눈으로 성능을 직접 확인합니다.

**4. 모델 내보내기 및 백업 (Step 11 ~ 13)**
- **ONNX Export**: 학습된 PyTorch 모델(`.pt`)을 실제 서비스나 다양한 플랫폼(C++, C#, Web 등)에서 사용하기 좋은 범용 포맷(`.onnx`)으로 변환합니다.
- **구글 드라이브 백업**: 런타임이 종료되어도 모델이 날아가지 않도록, 최종 학습된 모든 가중치 파일과 ONNX 파일을 구글 드라이브 지정 경로로 안전하게 복사합니다.

### 📊 모델 성능 요약 (Validation & Test)

이번 파인튜닝 파이프라인에서 학습된 세 가지 모델의 **mAP50 (Box / Mask)** 결과는 다음과 같습니다.

| 모델 (Model) | Validation mAP50 (Box/Mask) | Test mAP50 (Box/Mask) |
|---|---|---|
| **Single-Stage** | 0.9928 / 0.9928 | 0.9888 / 0.9888 |
| **2-Stage (Stage 1)** | 0.9948 / 0.9948 | 0.9852 / 0.9852 |
| **2-Stage (Stage 2)** | 0.9933 / 0.9933 | 0.9852 / 0.9852 |

> *모든 모델이 0.98 이상의 높은 정확도를 보이고 있으며, Single-Stage 모델이 Test 데이터셋에서 가장 균형 잡힌 높은 성능을 기록했습니다.*

### 💡 사용 팁 (Tip)
- 처음 실행하실 때는 **2단계 파인튜닝 활성화** 셀에서 `USE_TWO_STAGE = True`로 두고 끝까지 실행하여 모델 간 성능을 비교해 보세요.
- 클래스 이름이 다르다면 `2. 데이터셋 경로 설정` 셀에서 `CLASS_NAMES`를 수정하면 됩니다.
